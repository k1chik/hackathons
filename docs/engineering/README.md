## How to use these docs

- **Create the doc before writing code.** It forces clarity on what you're actually building.
- **Link to it from your PR description.** Reviewers need context.
- **Update it when decisions change.** A stale doc is worse than none.
- **Open Questions sections are deliberate.** Unresolved questions belong here, not buried in Slack.
